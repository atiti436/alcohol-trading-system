# 🎨 老闆UI需求規範

## 🎯 設計風格
**MUJI + 蘋果風格**：簡單、易懂、乾淨

### **視覺特色**
- **極簡主義**：去除不必要的裝飾
- **留白充足**：呼吸感，不擁擠
- **色彩克制**：以白、灰、黑為主，重點用色謹慎
- **字體清晰**：易讀性最優先
- **圓角設計**：溫和、友善的感覺

### **色彩規範**
```scss
// 主色調 - MUJI風格
$background: #FAFAFA;        // 淺灰背景
$card-background: #FFFFFF;   // 卡片白色
$text-primary: #2C2C2C;      // 深灰文字
$text-secondary: #8E8E93;    // 次要文字

// 功能色彩 - 蘋果風格
$success: #34C759;           // 蘋果綠
$warning: #FF9500;           // 蘋果橘
$error: #FF3B30;             // 蘋果紅
$info: #007AFF;              // 蘋果藍

// 商業色彩
$profit: #34C759;            // 獲利綠
$revenue: #007AFF;           // 營收藍
$pending: #FF9500;           // 待辦橘
```

## 📱 響應式重點

### **手機版優化**
- **字體大小**: 最小16px，確保老闆容易閱讀
- **觸控目標**: 按鈕最小44px×44px
- **重要資訊**: 待辦事項和業績數據優先顯示
- **操作便利**: 常用功能一鍵可達

### **平板版適應**
- **混合佈局**: 桌機和手機版的中間形態
- **雙欄設計**: 充分利用螢幕空間
- **觸控友善**: 支援手勢操作

## ⚠️ 安全顯示規則

### **角色權限UI差異**
```typescript
// 老闆(SUPER_ADMIN)看到的
interface BossUI {
  actualRevenue: number;    // 真實營收
  commission: number;       // 老闆抽成
  personalOrders: Order[];  // 個人調貨
  allCustomers: Customer[]; // 所有客戶
}

// 投資方(INVESTOR)看到的
interface InvestorUI {
  displayRevenue: number;   // 調整後營收
  commission: never;        // 完全隱藏
  personalOrders: never;    // 完全隱藏
  companyCustomers: Customer[]; // 只有投資項目客戶
}
```

### **價格顯示策略**
- **老闆**：顯示實際成本和真實售價
- **投資方**：顯示調整後的成本和顯示價格
- **UI提示**：投資方界面有「此為投資項目資料」提示
